91 void DepthmapDenoiseWeightedHuberImpl::computeSigmas(float epsilon,float theta){
92  /*
93  //This function is my best guess of what was meant by the line:
94  //"Gradient ascent/descent time-steps sigma_q , sigma_d are set optimally
95  //for the update scheme provided as detailed in [3]."
96  // Where [3] is :
97  //A. Chambolle and T. Pock. A first-order primal-dual
98  //algorithm for convex problems with applications to imaging.
99  //Journal of Mathematical Imaging and Vision, 40(1):120–
100  //145, 2011. 3, 4, 6
101  //
102  // I converted these mechanically to the best of my ability, but no
103  // explaination is given in [3] as to how they came up with these, just
104  // some proofs beyond my ability.
105  //
106  // Explainations below are speculation, but I think good ones:
107  //
108  // L appears to be a bound on the largest vector length that can be
109  // produced by the linear operator from a unit vector. In this case the
110  // linear operator is the differentiation matrix with G weighting
111  // (written AG in the DTAM paper,(but I use GA because we want to weight
112  // the springs rather than the pixels)). Since G has each row sum < 1 and
113  // A is a forward difference matrix (which has each input affecting at most
114  // 2 outputs via pairs of +-1 weights) the value is bounded by 4.0.
115  //
116  // So in a sense, L is an upper bound on the magnification of our step size.
117  //
118  // Lambda and alpha are called convexity parameters. They come from the
119  // Huber norm and the (d-a)^2 terms. The convexity parameter of a function
120  // is defined as follows:
121  // Choose a point on the function and construct a parabola of convexity
122  // c tangent at that point. Call the point c-convex if the parabola is
123  // above the function at all other points.
124  // The smallest c such that the function is c-convex everywhere is the
125  // convexity parameter.
126  // We can think of this as a measure of the bluntest tip that can trace the
127  // entire function.
128  // This is important because any gradient descent step that would not
129  // cause divergence on the tangent parabola is guaranteed not to diverge
130  // on the base function (since the parabola is always higher(i.e. worse)).
131  */
132 
133 
134  float lambda, alpha,gamma,delta,mu,rho,sigma;
135  float L=4;//lower is better(longer steps), but in theory only >=4 is guaranteed to converge. For the adventurous, set to 2 or 1.44
136 
137  lambda=1.0/theta;
138  alpha=epsilon;
139 
140  gamma=lambda;
141  delta=alpha;
142 
143  mu=2.0*std::sqrt(gamma*delta)/L;
144 
145  rho= mu/(2.0*gamma);
146  sigma=mu/(2.0*delta);
147 
148  sigma_d = rho;
149  sigma_q = sigma;
150 }
